# 小红书科技账号运营助手

一站式工具：账号分析 | 竞品拆解 | 选题灵感 | 自动发布 | 配图生成

# 一、账号分析

## 分析流程

1. 用浏览器打开账号主页
2. 采样最近 9-15 篇笔记：标题、点赞、收藏、评论、标签
3. 按五维评分，给出优化建议

## 五维评分

| 维度 | 关注点 |
|------|--------|
| 定位清晰度 | 科技细分领域是否明确 |
| 内容结构力 | 教程框架/代码示例是否清晰 |
| 互动转化力 | 收藏率（实用型内容核心） |
| 账号辨识度 | 人设/语气/视觉风格是否统一 |
| 增长可持续性 | 选题储备/热点追踪能力 |

## 输出格式

```
## 账号快照
- 定位：
- 内容支柱：
- 当前状态：

## 五维评分
- 定位清晰度：X/5
- 内容结构力：X/5
- 互动转化力：X/5
- 账号辨识度：X/5
- 增长可持续性：X/5

## 下步动作
1. 立刻调整...
2. 本周测试...
3. 持续观察...
```

```
- 标题骨架：
- 封面风格：
- 内容结构：

## 避免模仿
- 不适合的点：
```

# 三、选题灵感

## 科技选题公式

| 类型 | 公式 | 示例 |
|------|------|------|
| 测评型 | `用了X天XX，说说感受` | 用了30天Claude，说说真实感受 |
| 对比型 | `XX vs XX，谁更强` | Cursor vs Copilot，谁更强？ |
| 教程型 | `手把手教你用XX` | 手把手教你写AI提示词 |
| 盘点型 | `XX个XX必备` | 程序员5个必备工具 |
| 揭秘型 | `XX的隐藏功能` | ChatGPT的隐藏功能 |
| 踩坑型 | `XX的正确打开方式` | Git的正确打开方式 |

## 输出格式

```
1. 选题：（20字内）
   角度：
   人群：
   钩子：评论区问什么问题
   结构：开头/正文/结尾
```

# 四、自动发布

## 发布流程

1. 打开发布页 → 「上传图文」
2. 上传配图
3. 填写标题（≤20字）和正文
4. 添加话题标签
5. **停在发布按钮，等用户确认**

## 发布前检查

- [ ] 标题 ≤20 字
- [ ] 三要素齐全（封面/标题/正文）
- [ ] 无敏感词

## 配图生成命令

```bash
# 生成封面大字报
python scripts/render_xhs.py --cover "标题" -s ocean

# 生成多卡片
python scripts/render_xhs.py -t "标题" -c "正文..." -s terminal
```

### 配图主题

| 内容 | 风格 |
|------|------|
| AI工具 | `ocean` / `elegant` |
| 程序员 | `terminal` / `dark` |
| 数码测评 | `professional` |
| 效率工具 | `mint` / `botanical` |

# 五、自动回复评论

## 检查流程

1. 打开通知页 → 「评论和@」
2. 抓取最新评论，输出摘要
3. **等用户确认是否回复**

## 回复规则

1. 点击「回复」入口
2. 校验 placeholder 为 `回复 <用户名>`
3. 逐字输入（用 `type`，不用 `fill`）
4. 点「发送」按钮（不用 Enter）
5. 间隔 8-15 秒再发下一条

## 话术原则

- ≤280 字
- 不虚构经历
- 不隐性承诺
- 语气亲切自然

# 九、笔记复盘

## 复盘流程

1. 发布后立即记录初始数据
2. 在关键时间节点追踪数据
3. 生成复盘报告，分析优化点

## 追踪时间节点

| 时间点 | 意义 |
|--------|------|
| 1小时 | 初始流量分发验证 |
| 6小时 | 早中期数据预测 |
| 12小时 | 流量进入稳定期 |
| 24小时 | 完整自然流量周期 |
| 72小时 | 二次推荐效果 |
| 1周 | 完整生命周期 |

## 核心指标

| 指标 | 说明 | 优秀值 |
|------|------|--------|
| 互动率 | (点赞+收藏+评论)/浏览 | >5% |
| 收藏率 | 收藏/浏览 | >3% |
| 点赞率 | 点赞/浏览 | >2% |

## 复盘命令

```bash
# 首次记录笔记数据
python scripts/review_xhs.py --note-id xxx --record --api-mode

# 持续追踪（在各时间点自动记录）
python scripts/review_xhs.py --note-id xxx --track --daemon

# 查看历史数据
python scripts/review_xhs.py --note-id xxx --history

# 生成复盘报告
python scripts/review_xhs.py --note-id xxx --report

# 列出所有追踪的笔记
python scripts/review_xhs.py --list
```

## 优化建议

根据复盘数据自动生成：

1. **互动率低** → 优化标题封面，提高点击
2. **收藏率低** → 增加实操内容，提升实用价值
3. **增速放缓** → 考虑新笔记或互动活动激活
4. **早期流量低** → 测试不同发布时间
